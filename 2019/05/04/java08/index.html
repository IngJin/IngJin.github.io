JAVA기초 08. 메소드(Method)

작성일 2019-05-04 | Edited on 2019-05-06 | In JAVA | 댓글:

Method(메소드)란?

객체(클래스)가 가지고 있는 행위정보를 표현하는 것(동작, 기능) → 함수(Function)
동작(행위)를 만들어 놓고 필요할 때 사용하기 위해서 호출(Calling)
캡슐화의 원칙 : 클래스 안에 메소드를 구현, 하나의 클래스 안에는 여러개의 메소드가 존재

클래스(객체) 안에는 무엇이 들어가는가?

상태정보 : 멤버 변수(필드) → DTO Class, VO Class
행위정보 : 멤버 메소드 → DAO Class
main() : 자바는 반드시 main()메소드에서 프로그램이 동작(실행) → Main Class

메소드의 형식(메소드를 만드는 방법)

①public ②static ③void ④getSum(⑤int a, int b){ → 프로토타입 : 머리, 정의부
~메소드의 동작 구현~ ⑥return; → 바디 : 구현부
}

① 접근제어자 : public(접근허용), private(정보은닉 : 내부만 허용), protected, default
② 한정자 : static(자동으로 메모리에 생성 : 바로접근), 생략(객체를 생성해서 접근), final(재정의가 불가)
③ 리턴타입(반환값) : 메소드가 동작 후에 최종적으로 만들어내는 결과값의 데이터 타입
리턴을 하지 않는 경우 : void → 메소드 내부에 return 문이 없다.
리턴을 하는 경우 : 리턴되는 데이터의 타입 → 메소드 내부에 return 문이 있다.
④ 메소드 이름 : 소문자로 작성, 키워드는 사용이 불가, 의미있게 작성
⑤ 매개변수 리스트 : 메소드가 동작하기 위해서 외부로부터 입력되는 데이터 → 인수,인자,매개변수,parameter
⑥ return 문 : 메소드가 동작을 완료한 후, 결과값을 호출한 쪽으로 넘기는 행위

실인수(호출)와 가인수(정의)의 매개변수 전달 규칙 : Parameter Passing Rule

실인수와 가인수의 매개변수의 개수가 같아야 된다.
실인수와 가인수의 데이터 타입이 같아야 된다.
실인수와 가인수의 이름은 같아도 되고 달라도 된다.

메소드 오버로딩

클래스 안에 같은 이름을 가지고 있는 메소드
매개변수의 개수가 달라야한다.
매개변수의 개수가 같을 경우에는 데이터 타입이 달라야한다.
